# 直拳：技术指标实施计划（重心 + 发力顺序 + 拳面角度/轨迹 + 回收速度 + 步法协调）

## 1. 背景与约束

目标是从**单目视频**的 Pose33 骨架中，对 `标准动作要点/直拳.txt` 的关键技术指标逐条给出**是否合格**（默认不展示数字），并提供可选的“完整输出”用于排查。

已确认业务约束：
- 视频中包含**前手直拳（jab）**与**后手直拳（cross）**。
- 站姿**以左势（左脚前）为主**（允许少量例外）。
- 视角包含**正面与侧面**；侧面可能存在一定**斜角与俯仰角**（但不会特别夸张）。
- 输出需求：默认只给**每条指标是否合格**（不希望出现杂乱数字），但需要一个“完整输出”选项。
- 回收速度：若**出拳次数 N>=4** 且 **慢的占比>=75%**，则**仅该指标不合格**（不影响其它指标）。
- 重心结论：**侧面优先**（侧面可判定时以侧面为准；侧面不可判定时用正面兜底）。

## 2. 总体交付物

### 2.1 默认（干净输出）
- `tech_report.csv`
  - 每个视频一行
  - 每条指标一列：`合格 / 不合格 / 无法判定`
  - 每条指标附一句原因（不带数值）

### 2.2 可选（完整输出，用于排查）
- `tech_report_full.jsonl`
  - 包含阈值、有效帧统计、偏前/偏后统计、每次出拳事件与耗时等明细
- （可选）`*_debug_skeleton.mp4`
  - 叠加关键点、判定文字、关键量（用于人工抽检/对齐）

## 3. 数据输入与分段

### 3.1 Pose 提取
- 使用 MediaPipe PoseLandmarker（VIDEO mode）逐帧提取 Pose33：
  - 结构：`landmarks[T,33,4]`（x,y,z,visibility）
  - 同时计算每帧 `pose_view_score()`（用于正/侧切分）

### 3.2 正/侧切分（仅在“混合视角长视频”启用）
- 输入可能是：
  - 单视角视频（纯正面 / 纯侧面）
  - 混合视角长视频（前半正面、后半侧面等）
- 切分策略（保守，避免误切）：
  1) 平滑 view score（滑动均值）
  2) 以中位数二分为高/低
  3) 仅当两段都足够长、且均值差异足够大时，判定为 `mixed` 并输出 `front_seg/side_seg`
  4) 否则判定为 `single`

建议提供强制视角参数（避免误切造成侧面缺失）：
- `--view auto|side|front`

## 4. 指标总览与“评估窗口”

`直拳.txt` 指标包含：
- 发力顺序（蹬地→转腰→送肩）
- 拳面角度与轨迹
- 击打后迅速回收
- 身体重心稳定性
- 配合步法的协调性

为了避免“整段视频混入大量静止帧/走位帧”影响判定，建议所有与直拳动作相关的指标，统一基于“出拳事件窗口”评估：
- 先检测每次出拳（支持 jab/cross，左右臂都检测）
- 为每次出拳生成一个窗口：
  - `start = 伸直触发帧 - pre_pad`（例如 0.2s）
  - `peak = 伸直触发帧`
  - `end = 回收完成帧 + post_pad`（例如 0.1s）
- 需要整段统计的（如重心）也允许在“侧面段”全段统计，但仍可优先使用“出拳窗口聚合”作为更贴近动作的口径（待你确认）。

## 5. 重心检测（方案 1+2；侧面优先；方案 3 进阶）

### 5.1 关键原则：质量门控（宁可无法判定，不乱判）
- 规则计算必须满足关键点可见度（visibility）达到阈值（默认 `vis_thr=0.5`）
- 不满足的帧记为“无效帧”，不纳入合格/不合格统计
- 若有效帧数不足（默认 `min_valid_frames=15`）则输出 `无法判定`

### 5.2 侧面（主判定）：方案 2 强规则优先 + 方案 1 兜底

#### 4.2.1 方向统一（解决左侧拍/右侧拍）
目的：把“前方方向”统一成 `+x`，避免同一动作左右侧拍出现相反符号。

方法（逐帧或全段投票）：
- 用 `dir = sign(NOSE.x - hip_center.x)` 估计面朝方向
- 后续所有用于“前后”的 x 坐标都乘以 `dir`

若 nose 可靠性不足，可用髋/肩中心兜底；若仍不可靠，则侧面更多输出为 `无法判定`。

#### 4.2.2 前脚/后脚识别（左势优先 + 兜底纠正）
- 默认左势：前脚=左脚，后脚=右脚
- 兜底：统计该段中“哪只脚更靠前(+x)”的多数票，若与默认相反且差异明显，则自动切换

#### 4.2.3 方案 2（侧面规则，强判定）
来自动作要点（侧面）：
- 若**后脚大小腿夹角≈180°** → 重心偏前
- 若**前脚大小腿夹角≈180°** → 重心偏后
- 若**膝盖从上往下映射到的位置大约在脚背上** → 重心居中

工程化定义：
- 膝角：`angle(hip, knee, ankle)`，默认“≈180°”阈值：`knee_straight_thr=175°`
- 膝在脚背：`knee_x` 落在同侧脚的 `[heel_x, toe_x]` 的中间区间（默认 25%~75%）

优先级：
1) 后腿膝角直 → 偏前
2) 前腿膝角直 → 偏后
3) 膝在脚背 → 居中

#### 4.2.4 方案 1（侧面兜底）：髋部投影 + 支撑面比例
当方案 2 无强判定时启用。

定义：
- 髋部代理重心：`hip_x = mean(L_HIP.x, R_HIP.x)`
- 支撑面前后边界：两脚 heel/toe 合并后的 `[support_back_x, support_front_x]`
- 比例：`r = (hip_x - support_back_x) / (support_front_x - support_back_x)`
  - `r <= hip_r_back(默认0.35)` → 偏后
  - `r >= hip_r_front(默认0.65)` → 偏前
  - 其余 → 居中

#### 4.2.5 单段输出（侧面）
统计有效帧中：
- 偏前比例 `f_ratio`
- 偏后比例 `b_ratio`

判定（默认 `trigger_ratio=0.3`）：
- 若 `f_ratio >= trigger_ratio` → `不合格：重心偏前`
- 若 `b_ratio >= trigger_ratio` → `不合格：重心偏后`
- 若两者都不达标 → `合格：重心居中`
- 若两者都达标（冲突） → `无法判定：冲突（可能斜拍/遮挡）`

### 5.3 正面（兜底判定）：方案 2（前膝/前踝角）
来自动作要点（正面）：
- 前腿膝角≈180° → 重心偏后
- 前踝（小腿-脚背）夹角为锐角（<90°）→ 重心偏前

工程化定义（默认左势前腿=左腿）：
- 膝角：`angle(hip, knee, ankle)`，阈值 `knee_straight_thr=175°`
- 踝角：`angle(knee, ankle, foot_index)`，阈值 `ankle_acute_thr=90°`
- 单帧冲突（同时满足偏前与偏后）→ 该帧记为无效/冲突，不纳入有效统计

输出规则同侧面（占比触发）。

### 5.4 最终重心结论（侧面优先）
- 若侧面可判定（合格/不合格）→ 直接作为最终结论
- 否则使用正面结果兜底
- 若两者都无法判定 → 最终为 `无法判定`

## 6. 击打后迅速回收（侧面；N>=4 且慢>=75% 判该指标不合格）

本指标来自动作要点（侧面）：
- 从“击拳伸直”到“收手后大小臂夹角<45°”耗时 < 0.2s（按 30fps 约 6 帧）
- 若多次出拳慢的占比 >=75% 且 N>=4，则该指标判死（仅该指标不合格）

工程化实现建议：
- 对左右臂分别检测出拳事件（支持 jab/cross）：
  - 伸直事件：肘角 `>160°` 且 手腕相对肩在前方（沿 +x 的 `wrist_x - shoulder_x` > 阈值），连续 2 帧确认
  - 回收完成：肘角 `<45°` 连续 2 帧确认
  - 事件窗口：伸直到回收最大允许窗口 `max_window_sec=1.0`（窗口内未回收视为慢）
- 统计：
  - N：出拳事件总数（左右合并）
  - M：慢事件数（耗时>0.2s 或未回收）
- 判定：
  - 若 N < 4 → `无法判定：出拳次数不足`
  - 若 M/N >= 0.75 → `不合格：多数出拳回收超时`
  - 否则 → `合格`

## 7. 发力顺序（蹬地→转腰→送肩）

说明：单目视频无法直接测“真实力量链条”，这里采用“可解释的几何代理信号 + 时序关系”落地为合格/不合格/无法判定。

### 7.1 蹬地（后脚脚尖着地 + 脚底与地面夹角<90°）
来自 `直拳.txt`：
- 蹬地：后脚底与地面夹角小于90°，脚尖着地

推荐评估视角：侧面（优先），正面仅做参考。

工程化定义（侧面，假设地面在画面中近似水平；斜拍/俯仰明显时倾向无法判定）：
- “脚尖着地”近似：
  - 后脚 `FOOT_INDEX` 的 y 值接近该段“地面高度”（例如后脚 toe 的 y 接近窗口内 toe_y 的 90 分位）
  - 且 `HEEL.y` 明显高于 `TOE.y`（抬脚跟）
- “脚底与地面夹角”近似：
  - foot_vec = `HEEL - TOE`
  - ground_vec 近似为水平向量（或用双脚落点拟合地面方向）
  - foot_pitch = `angle(foot_vec, ground_vec)`，要求在一个合理范围（例如 10°~80°，避免噪声）

合格/不合格判定（建议基于“每次出拳窗口”的前半段）：
- 合格：在出拳开始前 `pre_pad` 时间内，出现满足上述条件的帧占比 ≥ `push_off_ratio`（初版建议 0.2~0.3）
- 不合格：几乎全程脚跟未抬/脚尖未稳定着地（占比 < 阈值）
- 无法判定：后脚关键点长期不可见、或地面方向无法稳定估计（例如俯仰过大）

### 7.2 转腰-送肩（上半身旋转一定度数）
来自 `直拳.txt`：
- 转腰-送肩：上半身旋转一定度数

单目视频下“转腰角度”属于弱可观测量，建议分两条代理指标：
- 转腰（正面更稳）：肩线与胯线的相对扭转
- 送肩（侧面更稳）：出拳侧肩相对胯中心的前送距离

#### 7.2.1 转腰（正面）
候选实现（2 选 1，建议先 1 后 2）：
1) 2D 扭转代理（不依赖 z）：
   - shoulder_angle = `atan2(R_SHOULDER.y - L_SHOULDER.y, R_SHOULDER.x - L_SHOULDER.x)`
   - hip_angle = `atan2(R_HIP.y - L_HIP.y, R_HIP.x - L_HIP.x)`
   - twist = `abs(wrap(shoulder_angle - hip_angle))`
   - 在“出拳窗口”内 twist 峰值 ≥ `twist_min_deg`
2) 3D 扭转代理（依赖 MediaPipe 的 z；更敏感但更贴近旋转）：
   - dz = `abs(L_SHOULDER.z - R_SHOULDER.z)` 或 `abs(L_HIP.z - R_HIP.z)`
   - 归一化后超过阈值

合格：twist 峰值达到阈值，且发生在“送肩/伸直”之前（时序见 7.3）
无法判定：肩/胯关键点长期不可见或画面严重斜拍导致信号不稳定

#### 7.2.2 送肩（侧面）
定义（侧面，统一前方为 +x）：
- punch_shoulder = 出拳侧的肩（L_SHOULDER 或 R_SHOULDER，随 jab/cross 自适应）
- hip_center_x = mean(L_HIP.x, R_HIP.x)
- shoulder_drive = (punch_shoulder.x - hip_center_x)（乘以 dir）

合格：在出拳窗口内 shoulder_drive 的增量峰值 ≥ `shoulder_drive_min`
无法判定：侧面段缺失或 shoulder/hip 可见度不足

### 7.3 发力顺序（时序一致性：蹬地→转腰→送肩→伸直）
为了落实“顺序”，对每次出拳事件提取 3 个关键时刻（峰值/首次满足阈值帧）：
- t_push：蹬地信号首次达标（后脚脚尖着地+抬脚跟）
- t_twist：转腰信号达到峰值或首次达标
- t_drive：送肩信号首次达标
- t_punch：出拳伸直时刻（事件 peak）

合格判定（初版）：
- t_push ≤ t_twist ≤ t_drive ≤ t_punch
- 允许轻微并行：t_twist 与 t_drive 可重叠（例如差值≤2~3帧）

无法判定：任一关键时刻缺失（信号不足/遮挡）

## 8. 拳面角度（轨迹暂不评估）

来自 `直拳.txt`：
- 手背与小臂成180°或稍微下弯（5°以内）

### 8.1 拳面角度（腕关节对齐）
推荐视角：侧面与正面均可，但应在“出拳伸直附近”评估。

工程化定义（Pose33 可用的手部点：WRIST / INDEX / PINKY / THUMB）：
- forearm_vec = `WRIST - ELBOW`
- hand_vec = `((INDEX + PINKY)/2) - WRIST`（更稳，避免单点抖动）
- wrist_align_angle = `angle(forearm_vec, hand_vec)`

合格阈值（对应“180°或下弯<=5°”）：
- `wrist_align_angle >= 175°`

评估时机：
- 在每次出拳事件的 peak 附近（例如 peak±2帧）取最大/中位数，避免单帧抖动

输出：
- 合格：大多数出拳事件满足对齐（比例 ≥ `wrist_ok_ratio`）
- 不合格：明显手腕塌陷/折腕（比例低于阈值）
- 无法判定：手部关键点可见度不足（Pose 的手点在低分辨率或遮挡时常不稳）

### 8.2 轨迹
本期**不评估轨迹**（按你的要求“轨迹不管”），因此在默认输出中该项不参与判定，也不会出现在“完整输出”里。

## 9. 配合步法的协调性（暂不评估）
本期**不评估步法协调**（按你的要求“步法协调不管”），因此该指标不纳入判定与输出。

## 10. 参数表（初版默认值）

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `vis_thr` | 0.5 | 关键点可见度阈值（低于则该帧无效） |
| `min_valid_frames` | 15 | 有效帧不足则无法判定 |
| `trigger_ratio` | 0.3 | 偏前/偏后占比触发不合格阈值 |
| `knee_straight_thr` | 175° | “膝角≈180°”判定阈值 |
| `ankle_acute_thr` | 90° | “踝角锐角”判定阈值 |
| `hip_r_back / hip_r_front` | 0.35 / 0.65 | 髋部比例 r 的偏后/偏前阈值 |
| `retract_sec_thr` | 0.2s | 回收耗时阈值 |
| `min_punches` | 4 | 出拳次数门槛（N>=4 才评估） |
| `fatal_slow_ratio` | 0.75 | 慢事件比例阈值（>=75% 判不合格） |
| `push_off_ratio` | 0.25 | 蹬地信号占比触发阈值（每次出拳窗口前半段） |
| `twist_min_deg` | 10° | 转腰最小扭转角（初版占位，需标定） |
| `shoulder_drive_min` | 0.03 | 送肩最小前送量（归一化前的占位，建议后续改为归一化量） |
| `wrist_ok_ratio` | 0.7 | 拳面角度在多次出拳中满足的比例阈值 |

阈值标定建议：
- 先用默认值跑通全流程
- 再用“标准合格视频”的分布做轻量标定（尤其是方案 1 的 r 分位数带宽）

## 11. 验证与验收

### 11.1 快速自检
- 对标准视频与少量人工确认的合格/不合格样本跑批处理
- 输出必须给出清晰的“合格/不合格/无法判定”以及原因短句

### 11.2 人工抽查（推荐）
- 随机抽取 10 段样本
- 通过 debug 骨架视频叠加（可选）确认：
  - 前脚/后脚识别是否合理
  - 侧面偏前/偏后判定与肉眼一致
  - 斜拍/遮挡时是否倾向输出“无法判定”而非乱判

## 12. 进阶方案 3（后续）

当 1+2 在“上身大幅前倾/后仰”或“髋部代理失真”场景仍不稳定时，引入：
- 分段人体质心 CoM（按人体分段质量比例加权）
- 将 CoM 投影到支撑面上，替代或增强髋部投影方案
