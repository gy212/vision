# 直拳技术指标：剩余任务清单与周排期（重心/发力顺序/拳面角度/回收速度）

## 0. 当前阶段结论（截至 2026-01-30）

已具备端到端批处理产出：
- 默认干净输出：`tech_report.csv`（合格/不合格/无法判定 + 不带数值的原因）
- 可选完整输出：`tech_report_full.jsonl`（包含事件、阈值、有效帧等明细）
- 可选调试视频：`debug_videos/*_debug.mp4`（骨架 + 指标状态 + 事件窗口提示）

覆盖指标：
- 重心稳定性（侧面优先；优先基于“出拳窗口”统计，兜底整段统计）
- 击打后迅速回收（侧面；N>=4 且慢>=75% 判死）
- 发力顺序（蹬地→转腰→送肩→伸直）的可解释代理版本（侧面为主）
- 拳面角度（腕-小臂对齐；多次出拳按比例触发）

## 1. 剩余模块优先级（按业务价值/风险/工作量综合）

### P0（必须做：决定可用性与误判率）
1) 阈值标定与数据验收闭环
- 目标：把“默认阈值”替换为基于样本分布的阈值，降低误判
- 依赖：标准合格样本、典型不合格样本、样本的“人工结论”
- 交付：阈值配置文件 + 标定报告（分布/命中率/失败案例）

2) 发力顺序：混合视角对齐（front twist + side drive 的同一次出拳关联）
- 目标：对“长视频前正后侧”等混合视角，做到同一出拳事件跨段关联
- 依赖：混合视角长视频样本、出拳次数足够（>=4）
- 交付：发力顺序指标稳定性提升 + full 输出增加跨段关联字段

3) 调试视频可用性提升
- 目标：调试视频能清晰定位“哪一次出拳/哪一段帧”触发不合格
- 依赖：抽检流程、可复现问题样本
- 交付：叠字更丰富（事件序号/关键阈值/有效帧），并支持只导出关键窗口片段（可选）

### P1（建议做：提升泛化与可解释性）
4) 重心进阶方案 3：分段质心 CoM（替代髋部代理的疑难场景）
- 目标：改善“上身大幅前倾/后仰、髋代理失真”导致的冲突/误判
- 依赖：若干疑难样本、人体分段权重方案（可固定表）
- 交付：CoM 投影版本的重心判定（可配置开关）+ 对比报告

5) jab/cross 区分与统计输出
- 目标：在 full 输出里对 jab/cross 分开统计事件与指标，便于教学反馈
- 依赖：左势为主的样本、少量右势样本用于兜底验证
- 交付：事件结构新增 `type=jab|cross` 与分项统计

### P2（可选：产品化与体验）
6) UI 接入（桌面端一键跑批/导出）
- 目标：在 Tkinter UI 中暴露 tech_eval 批处理入口与选项（view/pose/full/debug）
- 依赖：UI 流程确认、导出目录规范
- 交付：UI 增加“直拳技术指标评估”页/入口

## 2. 周级执行时间表（建议 4 周滚动）

> 责任人以“角色”表示：算法（Rule/Pose）、工程（CLI/产物/性能）、QA（抽检/标注）、产品（验收口径）。

### 第 1 周（2026-02-02 ~ 2026-02-08）：标定与验收闭环（P0-1）
- 责任人：QA（主）、算法（配合）、产品（确认口径）
- 依赖资源：
  - 合格样本 ≥ 10；不合格样本（偏前/偏后/回收慢/折腕/顺序错）各 ≥ 5
  - 每段样本给出“人工结论”（仅合格/不合格/不确定即可）
- 交付物：
  - `outputs/calibration_report.csv`：按指标统计命中率与无法判定比例
  - `docs/直拳_阈值标定报告.md`：阈值建议与失败案例截图/片段说明
  - `tech_eval.py` 中阈值调整（或抽离到配置文件）

### 第 2 周（2026-02-09 ~ 2026-02-15）：混合视角出拳关联（P0-2）
- 责任人：算法（主）、工程（配合）
- 依赖资源：
  - 混合视角长视频 ≥ 10（含正→侧、侧→正）
  - 出拳次数充足（建议每段 ≥ 4）
- 交付物：
  - 发力顺序：支持“跨段关联”的 full 明细字段（事件 id、段内帧号、关联置信度）
  - `tech_report_full.jsonl` 可追溯“本次不合格由第几次出拳触发”

### 第 3 周（2026-02-16 ~ 2026-02-22）：重心 CoM 进阶（P1-4）
- 责任人：算法（主）、QA（抽检）
- 依赖资源：
  - “髋代理失真”疑难样本集（前倾/后仰/斜拍）≥ 10
- 交付物：
  - CoM 分段质心实现（开关 + 配置）
  - 对比报告：髋代理 vs CoM（无法判定比例、冲突比例、与肉眼一致率）

### 第 4 周（2026-02-23 ~ 2026-03-01）：产品化收口（P0-3 + P2-6）
- 责任人：工程（主）、产品（主）、QA（验收）
- 依赖资源：
  - 抽检标准（通过/不通过/无法判定的口径）
  - 交付目录规范（CSV/JSONL/视频）
- 交付物：
  - 调试视频增强（关键窗口片段导出可选）
  - UI 接入（可选）：一键选择目录跑批 + 导出
  - 最终验收清单与回归流程文档

## 3. 验收口径（建议）

对每个指标，给出三类统计：
- 结果覆盖率：`无法判定` 比例（目标：可判定场景尽量低）
- 一致率：与人工结论一致的比例
- 可解释性：full 输出是否能定位到具体出拳事件/关键帧范围

## 4. 风险与对策

1) 单目 + 斜拍导致的可观测性弱
- 对策：质量门控更严格，宁可无法判定；增加 debug 视频辅助抽检

2) Pose 手部点不稳定导致拳面角度误判
- 对策：事件附近多帧取中位数；必要时降低该指标权重或提升“无法判定”倾向

3) 混合视角跨段关联复杂
- 对策：先落地“保守关联”（只在段切分边界附近关联），逐步扩大覆盖范围
